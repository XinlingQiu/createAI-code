 全栈智学 · AI 多模态教育大模型平台 (AI Learning Assistant)

## 📖 项目概述
本项目是一个基于大语言模型（LLM）与检索增强生成（RAG）技术的全栈智能教育辅助系统。系统旨在打破传统单向知识灌输模式，通过构建“学、练、测、评”一体化的闭环生态，为学习者提供具备私有知识融合、多轮上下文记忆、动态图谱生成及个性化路径规划能力的专属 AI 学习伴境。

## 🛠️ 核心技术栈
本项目采用前后端同构的现代化 Python Web 架构，深度整合了自然语言处理与多模态视觉技术：
* **前端渲染与状态管理**：`Streamlit` (实现高颜值 UI、复杂会话状态持久化与多页面路由)
* **数据处理与可视化**：`Pandas`, `Matplotlib` (构建动态学习数据看板与趋势大盘)
* **大模型编排框架**：`LangChain` (基于 LCEL 架构，管理 Prompt 工程、对话记忆 Memory 与结构化 JSON 输出)
* **核心大语言模型**：阿里云通义千问 `Qwen-Max` (逻辑推理/组卷/图谱) & `Qwen-VL-Max` (多模态视觉解析)
* **向量检索引擎 (RAG)**：`ChromaDB` + `BAAI/bge-small-zh-v1.5` (实现多格式私有语料的切分、嵌入与高维相似度检索)
* **知识图谱渲染**：`NetworkX` (计算节点引力拓扑布局并可视化多维课程关联)

## ✨ 核心功能模块

1. **📊 动态实时可视化工作台**
   - 实时追踪用户交互（问答、组卷、图谱生成等），动态计算学习时长与学科重心。
   - 渲染直观的 7 日学习趋势柱状图与全链路行为追踪明细表。
2. **💬 RAG 智能问答与知识讲解**
   - 挂载本地向量知识库，消除大模型幻觉。
   - 支持多轮上下文连贯追问，内置“一键体系化教案生成”与“课堂对话智能总结”功能。
3. **📷 Qwen-VL 多模态视觉答疑**
   - 突破纯文本限制，支持本地上传或摄像头实时捕获。
   - 瞬间 OCR 识别并解析复杂数学公式、结构图表与手写笔记。
4. **📂 知识库全生命周期管理**
   - 支持 `.pdf`, `.docx`, `.txt`, `.mp4` 多模态数据源一键注入与自动向量化（Embedding）。
   - 提供细粒度的单文件定点剥离与全库物理级格式化重载机制。
5. **✍️ 动态组卷与自动化批改**
   - 支持“细粒度专项练习”与“高仿真全真模拟卷”双模式调度。
   - 基于强约束 Prompt 生成结构化试卷，实现客观题自动判卷与主观题采分点对齐。
6. **🧠 知识图谱与课程多维导航**
   - 自动化降维拆解核心脉络，提取学科宏观章节目录。
   - 渲染带有高频考点、易错盲区靶向标记的动态 NetworkX 拓扑图，并智能化推演学习路径。
7. **🗓️ LLM 个性化学习规划**
   - 基于 Agent 代理，依据目标、时长与执行流派，推演生成带时间粒度的量化学习报告。
8. **📌 AI 靶向错题诊断本**
   - 自动接管测试模块的异常错题，利用大模型算力执行底层断点剖析，并生成等效镜像变形题。

## 🚀 安装与运行指南

### 1. 环境准备
确保您的计算机已安装 **Python 3.8+**（推荐 3.10 及以上）。

### 2. 克隆/下载项目
将项目文件夹下载至本地，并在终端（Terminal / CMD）中进入该项目根目录。

### 3. 安装核心依赖
建议在虚拟环境中运行，执行以下命令一键安装所有依赖包：
```bash
pip install streamlit pandas networkx matplotlib langchain langchain-openai langchain-community chromadb sentence-transformers docx2txt
4. API 环境变量配置
系统强依赖大模型 API，请在 app.py 与 rag_engine.py 中定位到以下变量，并替换为您申请的真实 API Key：

Python
ALIYUN_API_KEY = "sk-xxxxxxxxxxxxxxxxxxxxxxxx"
5. 启动系统
在终端中执行以下命令唤醒控制台：

Bash
streamlit run app.py
终端运行成功后，默认浏览器将自动弹出系统认证页面。

📁 目录结构说明
app.py：前端交互大厅、路由调度与系统主控入口。

rag_engine.py：RAG 检索链构建、多轮对话 Memory 管理引擎。

knowledge_builder.py：多模态数据加载器、文本切分与 Chroma 向量库 CRUD 核心逻辑。

uploads/：用户上传语料的本地物理缓存区（自动生成）。

chroma_db/：Chroma 向量数据库持久化挂载目录（自动生成）。
